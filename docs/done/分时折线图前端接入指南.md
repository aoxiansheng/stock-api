# 分时折线图前端接入指南

更新时间：2026-03-15

## 1. 适用范围

本文用于前端接入以下能力：

- `POST /api/v1/chart/intraday-line/snapshot`
- `POST /api/v1/chart/intraday-line/delta`
- `POST /api/v1/chart/intraday-line/release`
- 可选 WebSocket 事件 `chart.intraday.point`（兼容 `data`）

当前应以如下契约理解分时图：

- 首屏永远由 `snapshot` 提供。
- 后续更新由 `delta` 和可选 `WS` 提供。
- 当前后端首屏基线仍然是 `1m` 历史，后续增量可能带秒级时间戳。
- 前端主分时图应按“分钟桶”渲染，不要把每个秒级增量直接追加成新的 x 轴点。
- `snapshot.reference` 已提供昨收、今开和市场语义，前端直接消费，不要自行再调别的接口重算。

注意：

- 服务端已在 `snapshot` / `delta` 内部自动确保对应 `symbol` 的上游实时订阅。
- 前端不需要再额外建立一条 WebSocket 给后端“喂数”。
- 如果前端希望直接接收 `chart.intraday.point` 推送，而不是只靠 `delta`，仍然需要自己建立前端 WebSocket 连接。
- `chart.intraday.point` 是服务端广播事件名，不是前端传入的 `wsCapabilityType`。
- 若前端启用了 WS，退出页面时通常需要先 `unsubscribe`，再调用 `release`。

## 2. 接入结论

### 2.1 主图渲染原则

- 主分时图按分钟渲染。
- `snapshot` 返回的分钟历史点可直接落图。
- `delta` / `WS` 返回的秒级点，统一归并到“当前分钟桶”。
- 同一分钟内，价格使用最新价覆盖。
- 如果未来前端要展示量能，可以在同一分钟内累加 `volume`；当前前端不消费 `volume` 时可忽略。

例子：

- `05:19:38`、`05:19:41`、`05:19:57` 都归并到 `05:19:00` 这一分钟桶。
- 到 `05:20:00` 才正式产生新的主图点。

### 2.2 基准价原则

- 昨收基准线使用 `snapshot.reference.previousClosePrice`
- 今日开盘参考价使用 `snapshot.reference.sessionOpenPrice`
- 价格涨跌额/涨跌幅以前收为基准计算
- 前端不要自行按市场推导昨收/今开

### 2.3 市场语义原则

- `US` / `HK` / `CN`：`marketSession = regular`
- `CRYPTO`：`marketSession = utc_day`
- 时区以 `snapshot.reference.timezone` 为准，不要前端硬编码推断

## 3. 接入前提

1. 已具备读权限认证信息（API Key 或 JWT）。
2. 页面内维护唯一“分时会话状态”：

- `cursor`
- `pointsByMinute`
- `reference`
- `deltaPolling`
- `wsConnected`（仅启用前端 WS 时需要）

3. 若主图按分钟展示，建议直接维护 `Map<minute, point>`，不要把主状态设计成 `Map<second, point>`。

## 4. 协议速览

### 4.1 Snapshot

- 路径：`POST /api/v1/chart/intraday-line/snapshot`
- 作用：首屏一次性获取当前可得分时点位与参考价。
- 当前实现：`1m` 历史基线 + 当前已到达的实时窗口。
- 服务端行为：收到请求后会内部自动确保对应 symbol 的实时订阅存在。

必要请求字段：

```json
{
  "symbol": "AAPL.US",
  "provider": "infoway",
  "pointLimit": 30000
}
```

关键响应字段：

- `data.line.points[]`
- `data.sync.cursor`
- `data.sync.lastPointTimestamp`
- `data.capability.snapshotBaseGranularity`
- `data.capability.supportsFullDay1sHistory`
- `data.reference.previousClosePrice`
- `data.reference.sessionOpenPrice`
- `data.reference.marketSession`
- `data.reference.timezone`
- `data.reference.status`
- `data.metadata.realtimeMergedPoints`

参考响应片段：

```json
{
  "line": {
    "symbol": "AAPL.US",
    "market": "US",
    "tradingDay": "20260313",
    "granularity": "1s",
    "points": [
      {
        "timestamp": "2026-03-13T14:30:00.000Z",
        "price": 255.48,
        "volume": 0
      }
    ]
  },
  "capability": {
    "snapshotBaseGranularity": "1m",
    "supportsFullDay1sHistory": false
  },
  "reference": {
    "previousClosePrice": 255.76,
    "sessionOpenPrice": 255.48,
    "priceBase": "previous_close",
    "marketSession": "regular",
    "timezone": "America/New_York",
    "status": "complete"
  },
  "sync": {
    "cursor": "base64-signed-cursor",
    "lastPointTimestamp": "2026-03-13T14:30:00.000Z",
    "serverTime": "2026-03-13T14:30:01.120Z"
  }
}
```

重点说明：

- `line.granularity = "1s"` 不代表前端主图必须按秒渲染。
- 前端是否按分钟画，应该看你的图表语义，而不是机械按返回点的最细时间戳落点。
- 当前真正的首屏基线来源看 `capability.snapshotBaseGranularity`，现在是 `1m`。
- `supportsFullDay1sHistory=false` 时，不要向用户承诺“完整全日秒级历史回放”。

### 4.2 Snapshot.reference 的前端语义

字段说明：

- `previousClosePrice`：昨收参考价
- `sessionOpenPrice`：今开参考价
- `priceBase`：当前固定为 `previous_close`
- `marketSession`：`regular | utc_day`
- `timezone`：市场时区
- `status`：`complete | partial | unavailable`

状态处理建议：

- `complete`：正常展示昨收线、今开价、涨跌额、涨跌幅
- `partial`：能展示哪个值就展示哪个值，缺失项降级隐藏
- `unavailable`：仅展示价格曲线，不展示昨收线/今开/涨跌幅

### 4.3 Delta

- 路径：`POST /api/v1/chart/intraday-line/delta`
- 作用：仅拉取 `cursor` 之后的增量点。
- 当前角色：在 HTTP-only 模式下，`delta` 是持续加载未来点位的主通道。
- 服务端行为：收到请求后会内部自动确保对应 symbol 的实时订阅存在。
- 必须传 `cursor`。

```json
{
  "symbol": "AAPL.US",
  "provider": "infoway",
  "cursor": "base64-signed-cursor",
  "limit": 2000,
  "strictProviderConsistency": false
}
```

关键响应字段：

- `data.delta.points[]`
- `data.delta.nextCursor`
- `data.delta.hasMore`

前端消费原则：

- `delta.points[]` 中可能出现秒级时间戳。
- 如果主图按分钟渲染，应先归并到对应分钟桶，再刷新图表。
- 不要把 `05:19:38`、`05:19:39`、`05:19:40` 直接画成三个主图点。

推荐策略：

- 页面活跃时按 `1s` 轮询 `delta`
- 遇到 `CURSOR_EXPIRED` 时立即重拉 `snapshot`

### 4.4 WebSocket（可选增强）

- 连接地址：`/api/v1/stream-receiver/connect`（Socket.IO）
- 订阅事件：`subscribe`
- 订阅时仍使用 `wsCapabilityType=stream-stock-quote`
- 服务端会额外广播统一后的分时领域事件 `chart.intraday.point`
- 停止接收推送时，前端应发送 `unsubscribe`

订阅参数建议：

```json
{
  "symbols": ["AAPL.US"],
  "wsCapabilityType": "stream-stock-quote",
  "preferredProvider": "infoway"
}
```

接收事件优先级：

1. `chart.intraday.point`
2. `data`

前端消费原则：

- WS 是低延迟增强通道，不是分时图可用性的必要前提
- 即使接了 WS，主图仍建议按分钟桶更新
- WS 到来的秒级点只更新当前分钟桶
- `delta` 仍建议保留为补洞和恢复接口

### 4.5 Release

- 路径：`POST /api/v1/chart/intraday-line/release`
- 作用：显式释放由分时图接口自动拉起的内部实时订阅
- 不负责：替当前前端 WebSocket 连接执行 `unsubscribe`

```json
{
  "symbol": "AAPL.US",
  "provider": "infoway",
  "market": "US"
}
```

注意：

- 若当前没有活跃内部订阅，`released=false` 属于正常结果
- 如果前端在 `release` 后继续调用 `delta`，服务端会重新自动拉起订阅
- 如果前端启用了 WS，推荐顺序是先 `unsubscribe`，再调用 `release`

## 5. 市场语义

### 5.1 US / HK / CN

- `marketSession = regular`
- 昨收、今开由后端按对应市场交易日语义计算
- 前端不需要也不应该自己处理夏令时、午休、集合竞价等细节
- 展示时间轴时以 `reference.timezone` 为准

### 5.2 CRYPTO

- `marketSession = utc_day`
- 昨收和今开按 UTC 自然日计算
- 例如 `BTCUSDT` 的日切换锚点是 `00:00:00 UTC`

## 6. 推荐接入模式

### 6.1 基础模式：HTTP-only

适用场景：

- 普通分时图展示
- 可以接受 `1s` 级数据同步延迟
- 希望前端实现最简单

推荐时序：

1. 页面加载后调用一次 `snapshot`
2. 拿到 `points`、`reference`、`cursor` 后立即渲染首屏
3. 启动 `delta` 轮询，持续消费未来点位
4. 若收到 `CURSOR_EXPIRED`，立即重拉 `snapshot`
5. 页面卸载或切换 symbol 时调用 `release`
6. 任意时刻禁止每秒重拉 `snapshot`

### 6.2 增强模式：HTTP + WS

适用场景：

- 需要更低的前端展示延迟
- 希望优先走事件推送更新 UI
- 仍需要 `delta` 做恢复兜底

推荐时序：

1. 页面加载后调用一次 `snapshot`
2. 渲染首屏
3. 建立前端 WS，发送 `subscribe + wsCapabilityType=stream-stock-quote`
4. WS 正常时，可暂停高频 `delta`
5. WS 断开时，立即启用 `delta` `1s` 轮询补洞
6. WS 恢复稳定后，可停止高频 `delta`
7. 页面卸载或切换 symbol 时，先发送 `unsubscribe`
8. 随后调用 `release`

## 7. 前端状态设计

推荐类型：

```ts
type IntradayPoint = {
  timestamp: string;
  price: number;
  volume: number;
};

type IntradayReference = {
  previousClosePrice: number | null;
  sessionOpenPrice: number | null;
  priceBase: "previous_close";
  marketSession: "regular" | "utc_day";
  timezone: string;
  status: "complete" | "partial" | "unavailable";
};

type MinuteLinePoint = {
  timestamp: string;
  price: number;
  volume: number;
};

type IntradayState = {
  symbol: string;
  provider: string;
  cursor: string;
  reference: IntradayReference | null;
  pointsByMinute: Map<number, MinuteLinePoint>;
  deltaTimer: ReturnType<typeof setTimeout> | null;
  deltaInFlight: boolean;
};
```

说明：

- 主图状态维护 `pointsByMinute`
- `cursor` 仍然按后端返回原样保存
- `reference` 单独存储，供基准线和涨跌信息直接消费

## 8. 分钟桶归并规则

归并规则：

1. 先把任意返回点的时间戳归整到分钟桶
2. 同一分钟已有点时，使用最新价覆盖 `price`
3. 若使用 `volume`，则把同一分钟内的增量累加
4. 若前端当前不展示 `volume`，可以只覆盖 `price`

建议实现：

```ts
function toMinuteBucket(ts: string): number {
  const date = new Date(ts);
  date.setUTCSeconds(0, 0);
  return date.getTime();
}

function mergePointIntoMinuteBucket(
  pointsByMinute: Map<number, MinuteLinePoint>,
  point: IntradayPoint,
): void {
  const bucket = toMinuteBucket(point.timestamp);
  const existing = pointsByMinute.get(bucket);

  if (!existing) {
    pointsByMinute.set(bucket, {
      timestamp: new Date(bucket).toISOString(),
      price: point.price,
      volume: point.volume ?? 0,
    });
    return;
  }

  pointsByMinute.set(bucket, {
    timestamp: existing.timestamp,
    price: point.price,
    volume: (existing.volume ?? 0) + (point.volume ?? 0),
  });
}

function mergePoints(
  pointsByMinute: Map<number, MinuteLinePoint>,
  incoming: IntradayPoint[],
): void {
  for (const point of incoming) {
    mergePointIntoMinuteBucket(pointsByMinute, point);
  }
}

function toSortedSeries(
  pointsByMinute: Map<number, MinuteLinePoint>,
): MinuteLinePoint[] {
  return Array.from(pointsByMinute.entries())
    .sort((a, b) => a[0] - b[0])
    .map(([, point]) => point);
}
```

如果当前前端完全不消费 `volume`，也可以把更新逻辑简化成只维护 `price`。

## 9. 前端实现模板（TypeScript）

### 9.1 HTTP-only 基础模板

```ts
type SnapshotResponse = {
  line: {
    points: IntradayPoint[];
  };
  reference: IntradayReference;
  sync: {
    cursor: string;
  };
};

type DeltaResponse = {
  delta: {
    points: IntradayPoint[];
    hasMore: boolean;
    nextCursor: string;
  };
};

const API_BASE_URL = "http://127.0.0.1:3001";

async function postJson<T>(
  path: string,
  body: Record<string, unknown>,
  headers: Record<string, string>,
): Promise<T> {
  const resp = await fetch(`${API_BASE_URL}${path}`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      ...headers,
    },
    body: JSON.stringify(body),
  });

  const json = await resp.json();
  if (!resp.ok || json?.success !== true) {
    const err = new Error(json?.message || `HTTP ${resp.status}`);
    (err as any).payload = json;
    throw err;
  }

  return json.data as T;
}

async function loadSnapshot(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  const data = await postJson<SnapshotResponse>(
    "/api/v1/chart/intraday-line/snapshot",
    {
      symbol: state.symbol,
      provider: state.provider,
      pointLimit: 30000,
    },
    authHeaders,
  );

  state.pointsByMinute.clear();
  mergePoints(state.pointsByMinute, data.line.points || []);
  state.reference = data.reference;
  state.cursor = data.sync.cursor;
}

async function pullDeltaOnce(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  const data = await postJson<DeltaResponse>(
    "/api/v1/chart/intraday-line/delta",
    {
      symbol: state.symbol,
      provider: state.provider,
      cursor: state.cursor,
      limit: 2000,
      strictProviderConsistency: false,
    },
    authHeaders,
  );

  mergePoints(state.pointsByMinute, data.delta.points || []);
  state.cursor = data.delta.nextCursor;
}

function stopDeltaPolling(state: IntradayState): void {
  if (!state.deltaTimer) return;
  clearTimeout(state.deltaTimer);
  state.deltaTimer = null;
}

function startDeltaPolling(
  state: IntradayState,
  authHeaders: Record<string, string>,
  onSeries: (series: MinuteLinePoint[]) => void,
  intervalMs: number = 1000,
): void {
  if (state.deltaTimer) return;

  const tick = async () => {
    if (state.deltaInFlight) {
      state.deltaTimer = setTimeout(tick, intervalMs);
      return;
    }

    state.deltaInFlight = true;
    try {
      await pullDeltaOnce(state, authHeaders);
      onSeries(toSortedSeries(state.pointsByMinute));
    } catch (error: any) {
      const message = String(error?.payload?.message || error?.message || "");
      if (message.includes("CURSOR_EXPIRED")) {
        await loadSnapshot(state, authHeaders);
        onSeries(toSortedSeries(state.pointsByMinute));
      }
    } finally {
      state.deltaInFlight = false;
      state.deltaTimer = setTimeout(tick, intervalMs);
    }
  };

  state.deltaTimer = setTimeout(tick, intervalMs);
}
```

### 9.2 可选 WS 增强模板

```ts
import { io, Socket } from "socket.io-client";

type WsIntradayState = IntradayState & {
  ws: Socket | null;
  wsConnected: boolean;
};

function connectWs(
  state: WsIntradayState,
  auth: { appKey: string; accessToken: string },
  onSeries: (series: MinuteLinePoint[]) => void,
  onDisconnect: () => void,
  onReconnectStable: () => void,
): void {
  const socket = io(API_BASE_URL, {
    path: "/api/v1/stream-receiver/connect",
    transports: ["websocket"],
    auth: {
      appKey: auth.appKey,
      accessToken: auth.accessToken,
    },
  });

  state.ws = socket;

  socket.on("connect", () => {
    state.wsConnected = true;
    socket.emit("subscribe", {
      symbols: [state.symbol],
      wsCapabilityType: "stream-stock-quote",
      preferredProvider: state.provider,
    });
    onReconnectStable();
  });

  socket.on("chart.intraday.point", (payload: any) => {
    const point = payload?.point;
    const cursor = payload?.cursor;
    if (!point?.timestamp) return;

    mergePoints(state.pointsByMinute, [point]);

    if (typeof cursor === "string" && cursor.length > 0) {
      state.cursor = cursor;
    }

    onSeries(toSortedSeries(state.pointsByMinute));
  });

  socket.on("disconnect", () => {
    state.wsConnected = false;
    onDisconnect();
  });
}

async function disconnectWs(state: WsIntradayState): Promise<void> {
  const socket = state.ws;
  if (!socket) return;

  await new Promise<void>((resolve) => {
    let resolved = false;

    const finish = () => {
      if (resolved) return;
      resolved = true;
      socket.off("unsubscribe-ack", handleAck);
      socket.off("unsubscribe-error", handleError);
      resolve();
    };

    const handleAck = () => finish();
    const handleError = () => finish();

    socket.once("unsubscribe-ack", handleAck);
    socket.once("unsubscribe-error", handleError);

    socket.emit("unsubscribe", {
      symbols: [state.symbol],
      wsCapabilityType: "stream-stock-quote",
    });

    setTimeout(finish, 1000);
  });

  socket.disconnect();
  state.ws = null;
  state.wsConnected = false;
}
```

## 10. UI 直接消费建议

### 10.1 昨收线

- 基准线值：`reference.previousClosePrice`
- 仅当 `reference.status !== "unavailable"` 且值非空时展示

### 10.2 今开价

- 显示值：`reference.sessionOpenPrice`
- 仅当值非空时展示

### 10.3 涨跌额 / 涨跌幅

建议计算：

```ts
function buildPriceSummary(
  lastPrice: number | null,
  reference: IntradayReference | null,
) {
  const base = reference?.previousClosePrice ?? null;
  if (lastPrice == null || base == null) {
    return {
      change: null,
      changePercent: null,
    };
  }

  const change = lastPrice - base;
  const changePercent = base === 0 ? null : change / base;

  return {
    change,
    changePercent,
  };
}
```

### 10.4 时区展示

- 横轴时间格式化建议使用 `reference.timezone`
- `CRYPTO` 应按 `UTC` 展示交易日语义

## 11. 错误处理与恢复策略

1. `400 INVALID_ARGUMENT`

- 检查请求体是否传了非白名单字段
- 当前不要传 `includePrePost`、`since`

2. `409 CURSOR_EXPIRED`

- 丢弃旧 `cursor`
- 立即重拉 `snapshot`
- 然后继续 `delta` 或等待 WS 推送

3. `503 PROVIDER_UNAVAILABLE`

- 前端提示“数据源暂不可用”
- 可保留最近一次图形并展示刷新按钮

4. 可选前端 WS 断连

- 启动 `delta` `1s` 轮询补洞
- WS 恢复后可停止高频轮询

5. `reference.status = partial`

- 能展示哪个值就展示哪个值
- 缺失的昨收/今开字段不要自行补算

6. `reference.status = unavailable`

- 允许只展示曲线
- 隐藏昨收线、今开、涨跌幅模块

7. `realtimeMergedPoints = 0`

- 不要直接判定失败
- 这通常表示当前响应尚未并入实时点，继续 `delta` 即可

8. `release`

- 若启用了前端 WS，先执行 `unsubscribe`，再调用 `release`
- `release` 只负责内部上游订阅，不替代前端 WS 退订

## 12. 实现细节建议

1. 主图使用分钟桶，避免“分钟首屏 + 秒级后续”导致图表撕裂。
2. `cursor` 视为不透明字符串，不要解码、拼接或篡改。
3. `reference` 由后端统一给出，前端不要自己再调日 K 或其他接口计算。
4. `supportsFullDay1sHistory=false` 时，不要把当前接口包装成“全日秒级回放”。
5. 不要把“额外手动拉一条 WS 给后端喂数”作为接入前提。
6. 如果当前前端不消费 `volume`，可以忽略量的累加，但保留字段兼容性更稳妥。

## 13. 联调检查清单

1. `snapshot` 返回后是否拿到 `cursor`、`reference`。
2. `reference.previousClosePrice`、`reference.sessionOpenPrice` 是否可直接用于 UI。
3. `US/HK/CN` 是否收到 `marketSession=regular`。
4. `CRYPTO` 是否收到 `marketSession=utc_day` 且 `timezone=UTC`。
5. 不建立前端 WS 时，`delta` 连续轮询是否仍能持续拿到未来点位。
6. 建立前端 WS 时，是否以 `wsCapabilityType=stream-stock-quote` 订阅成功，并能收到 `chart.intraday.point`。
7. 秒级增量是否被正确归并到当前分钟桶，而不是直接追加成新的主图点。
8. `CURSOR_EXPIRED` 时是否能自动重拉 `snapshot` 恢复。
9. 页面卸载或切换 symbol 时，若启用了 WS，是否先 `unsubscribe`，再 `release`。
10. 是否完全避免“每秒 `snapshot` 全量请求”。
