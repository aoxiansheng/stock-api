# 分时折线图前端接入指南

更新时间：2026-03-14

## 1. 适用范围

本文用于前端接入以下能力：

- `POST /api/v1/chart/intraday-line/snapshot`
- `POST /api/v1/chart/intraday-line/delta`
- `POST /api/v1/chart/intraday-line/release`
- 可选 WebSocket 事件 `chart.intraday.point`（兼容 `data`）

当前应以如下契约理解分时图：

- **基础接入模式：`Snapshot` 首次全量 + `Delta` 持续增量**
- **增强接入模式：`Snapshot` 首次全量 + 可选 `WS` 低延迟推送 + `Delta` 补洞恢复**

注意：

- 服务端已在 `snapshot` / `delta` 内部自动确保对应 `symbol` 的上游实时订阅。
- 前端**不需要**再额外建立一条 WebSocket 来为分时图接口“喂数”或触发后端拉流。
- 如果前端希望直接接收 `chart.intraday.point` 推送，而不是通过 `delta` 轮询获取未来增量，则仍然需要自己建立前端 WebSocket 连接。
- `chart.intraday.point` 是服务端广播的事件名，不是前端传入的 `wsCapabilityType`。
- 若前端启用了 WS，则退出页面时通常需要做两步清理：先发 WebSocket `unsubscribe` 停止“下游到我们”的推送，再调 `release` 释放“我们到上游”的内部实时订阅。

## 2. 接入前提

1. 已具备读权限认证信息（API Key 或 JWT）。
2. 页面内有唯一“分时会话状态”：
- `cursor`
- `points`
- `deltaPolling`
- `wsConnected`（仅在启用前端 WS 增强时需要）
3. 建议将 `points` 维护为 `Map<second, point>`，渲染前再排序转数组。

## 3. 协议速览

### 3.1 Snapshot

- 路径：`POST /api/v1/chart/intraday-line/snapshot`
- 作用：首屏一次性获取当前可得分时点位。
- 当前实现：`1m` 历史基线 + 当前已到达的实时 `1s` 点位窗口。
- 服务端行为：收到请求后会内部自动确保对应 symbol 的实时订阅存在。

必要请求字段：

```json
{
  "symbol": "AAPL.US",
  "provider": "infoway",
  "pointLimit": 30000
}
```

关键响应字段（位于统一响应包装 `data` 内）：

- `data.line.points[]`
- `data.sync.cursor`
- `data.sync.lastPointTimestamp`
- `data.capability.supportsFullDay1sHistory`
- `data.metadata.realtimeMergedPoints`（诊断字段，可用于观测本次是否已并入实时点）

注意：

- 首次 `snapshot` 返回时，`realtimeMergedPoints` 可能仍为 `0`，这不一定代表异常。
- 即使首次 `snapshot` 仍以历史基线为主，后续 `delta` 也会继续消费未来点位。

### 3.2 Delta

- 路径：`POST /api/v1/chart/intraday-line/delta`
- 作用：仅拉取 `cursor` 之后的增量点。
- 当前角色：在 HTTP-only 模式下，`delta` 就是持续加载未来分时点的主通道。
- 服务端行为：收到请求后会内部自动确保对应 symbol 的实时订阅存在。
- 必须传 `cursor`。

```json
{
  "symbol": "AAPL.US",
  "provider": "infoway",
  "cursor": "base64-signed-cursor",
  "limit": 2000,
  "strictProviderConsistency": false
}
```

关键响应字段：

- `data.delta.points[]`
- `data.delta.nextCursor`
- `data.delta.hasMore`

推荐策略：

- 页面活跃时按 `1s` 轮询 `delta`。
- 遇到 `CURSOR_EXPIRED` 时立即重拉 `snapshot`，然后继续轮询。

### 3.3 WebSocket（可选增强）

- 连接地址：`/api/v1/stream-receiver/connect`（Socket.IO）
- 订阅事件：`subscribe`
- 订阅语义：
- 前端向服务端发送 `subscribe`，其中 `wsCapabilityType` 仍应使用 `stream-stock-quote`。
- 服务端在股票流能力上完成订阅后，会额外广播统一后的分时领域事件 `chart.intraday.point`。
- `chart.intraday.point` 不是单独的 WebSocket capability，不能作为 `wsCapabilityType` 传入。
- 前端如果要停止接收推送，应向当前 Socket 发送 `unsubscribe`。
- 订阅参数建议：

```json
{
  "symbols": ["AAPL.US"],
  "wsCapabilityType": "stream-stock-quote",
  "preferredProvider": "infoway"
}
```

接收事件优先级：

1. `chart.intraday.point`（推荐，分时领域事件）
2. `data`（兼容原始流事件）

注意：

- 前端 WS 是**低延迟增强通道**，不是分时图可用性的必要前置条件。
- 即使前端不接 WS，只要持续调用 `delta`，也可以持续拿到未来增量。
- 即使前端接了 WS，`delta` 仍然建议保留为补洞和恢复接口。
- 正确理解应为：订阅 `stream-stock-quote`，监听 `chart.intraday.point`。
- `unsubscribe` 只负责停止当前前端连接继续接收推送，不负责释放分时图接口内部自动拉起的上游订阅。

### 3.4 Release

- 路径：`POST /api/v1/chart/intraday-line/release`
- 作用：显式释放由分时图接口自动拉起的内部实时订阅。
- 不负责：替当前前端 WebSocket 连接执行 `unsubscribe`。
- 推荐调用时机：
- 页面卸载
- 切换 symbol 前
- 长时间离开分时图页面时

```json
{
  "symbol": "AAPL.US",
  "provider": "infoway",
  "market": "US"
}
```

关键响应字段：

- `data.release.released`
- `data.release.symbol`
- `data.release.provider`
- `data.release.wsCapabilityType`

注意：

- 若当前没有活跃内部订阅，`released=false` 属于正常结果。
- 如果前端在 `release` 后继续调用 `delta`，服务端会重新自动拉起订阅。
- 如果前端启用了 WS，推荐顺序是先 `unsubscribe`，再调用 `release`。

### 3.5 退出时的两段清理

若页面启用了前端 WS，退出分时图页面时需要理解为两段链路：

1. 前端 -> 我们的 WebSocket 服务
- 调用方式：发送 `unsubscribe`
- 作用：让当前前端连接不再收到 `chart.intraday.point`

2. 我们的分时图服务 -> 上游实时流能力
- 调用方式：`POST /api/v1/chart/intraday-line/release`
- 作用：释放分时图 API 内部自动拉起的上游实时订阅

因此：

- HTTP-only 模式只需要调用 `release`
- HTTP + WS 模式通常需要先 `unsubscribe`，再 `release`

## 4. 推荐接入模式

### 4.1 基础模式：HTTP-only

适用场景：

- 普通分时图展示
- 可以接受 `1s` 级更新
- 希望前端实现最简单

推荐时序：

1. 页面加载后调用一次 `snapshot`。
2. 拿到 `points` 和 `cursor` 后立即渲染首屏。
3. 启动 `delta` 轮询，持续消费未来点位。
4. 若收到 `CURSOR_EXPIRED`，立即重拉 `snapshot` 并替换状态。
5. 页面卸载或切换 symbol 时调用 `release`。
6. 任意时刻禁止“每秒重新拉 `snapshot` 全量”。

### 4.2 增强模式：HTTP + WS

适用场景：

- 需要更低的前端展示延迟
- 希望优先走事件推送更新 UI
- 仍需要 `delta` 做恢复兜底

推荐时序：

1. 页面加载后调用一次 `snapshot`。
2. 渲染首屏。
3. 建立前端 WS，发送 `subscribe + wsCapabilityType=stream-stock-quote`，并监听 `chart.intraday.point`。
4. WS 正常时，可暂停每秒 `delta` 轮询，只在断线、前后台切换恢复、或定期对账时触发 `delta`。
5. WS 断开时，立即启用 `delta` `1s` 轮询补洞。
6. WS 恢复稳定后，可停止高频 `delta` 轮询。
7. 页面卸载或切换 symbol 时，先发送 `unsubscribe` 停止前端推送。
8. 随后调用 `release` 释放分时图接口内部自动拉起的上游订阅。

## 5. 前端实现模板（TypeScript）

### 5.1 HTTP-only 基础模板

```ts
type IntradayPoint = {
  timestamp: string;
  price: number;
  volume: number;
};

type IntradayState = {
  symbol: string;
  provider: string;
  cursor: string;
  pointsBySecond: Map<number, IntradayPoint>;
  deltaTimer: ReturnType<typeof setTimeout> | null;
  deltaInFlight: boolean;
};

const API_BASE_URL = "http://127.0.0.1:3001";

function toSecondBucket(ts: string): number {
  return Math.floor(new Date(ts).getTime() / 1000) * 1000;
}

function upsertPoints(
  pointsBySecond: Map<number, IntradayPoint>,
  incoming: IntradayPoint[],
): void {
  for (const point of incoming) {
    pointsBySecond.set(toSecondBucket(point.timestamp), point);
  }
}

function toSortedSeries(pointsBySecond: Map<number, IntradayPoint>): IntradayPoint[] {
  return Array.from(pointsBySecond.entries())
    .sort((a, b) => a[0] - b[0])
    .map(([, point]) => point);
}

async function postJson<T>(
  path: string,
  body: Record<string, unknown>,
  headers: Record<string, string>,
): Promise<T> {
  const resp = await fetch(`${API_BASE_URL}${path}`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      ...headers,
    },
    body: JSON.stringify(body),
  });
  const json = await resp.json();
  if (!resp.ok || json?.success !== true) {
    const err = new Error(json?.message || `HTTP ${resp.status}`);
    (err as any).payload = json;
    throw err;
  }
  return json.data as T;
}

async function loadSnapshot(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  const data = await postJson<{
    line: { points: IntradayPoint[] };
    sync: { cursor: string };
  }>(
    "/api/v1/chart/intraday-line/snapshot",
    {
      symbol: state.symbol,
      provider: state.provider,
      pointLimit: 30000,
    },
    authHeaders,
  );

  state.pointsBySecond.clear();
  upsertPoints(state.pointsBySecond, data.line.points || []);
  state.cursor = data.sync.cursor;
}

async function pullDeltaOnce(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  const data = await postJson<{
    delta: {
      points: IntradayPoint[];
      hasMore: boolean;
      nextCursor: string;
    };
  }>(
    "/api/v1/chart/intraday-line/delta",
    {
      symbol: state.symbol,
      provider: state.provider,
      cursor: state.cursor,
      limit: 2000,
      strictProviderConsistency: false,
    },
    authHeaders,
  );

  upsertPoints(state.pointsBySecond, data.delta.points || []);
  state.cursor = data.delta.nextCursor;
}

function stopDeltaPolling(state: IntradayState): void {
  if (!state.deltaTimer) return;
  clearTimeout(state.deltaTimer);
  state.deltaTimer = null;
}

function startDeltaPolling(
  state: IntradayState,
  authHeaders: Record<string, string>,
  onSeries: (series: IntradayPoint[]) => void,
  intervalMs: number = 1000,
): void {
  if (state.deltaTimer) return;

  const tick = async () => {
    if (state.deltaInFlight) {
      state.deltaTimer = setTimeout(tick, intervalMs);
      return;
    }

    state.deltaInFlight = true;
    try {
      await pullDeltaOnce(state, authHeaders);
      onSeries(toSortedSeries(state.pointsBySecond));
    } catch (error: any) {
      const message = String(error?.payload?.message || error?.message || "");
      if (message.includes("CURSOR_EXPIRED")) {
        await loadSnapshot(state, authHeaders);
        onSeries(toSortedSeries(state.pointsBySecond));
      }
    } finally {
      state.deltaInFlight = false;
      state.deltaTimer = setTimeout(tick, intervalMs);
    }
  };

  state.deltaTimer = setTimeout(tick, intervalMs);
}
```

### 5.2 可选 WS 增强模板

```ts
import { io, Socket } from "socket.io-client";

type WsIntradayState = IntradayState & {
  ws: Socket | null;
  wsConnected: boolean;
};

function connectWs(
  state: WsIntradayState,
  auth: { appKey: string; accessToken: string },
  onSeries: (series: IntradayPoint[]) => void,
  onDisconnect: () => void,
  onReconnectStable: () => void,
): void {
  const socket = io(API_BASE_URL, {
    path: "/api/v1/stream-receiver/connect",
    transports: ["websocket"],
    auth: {
      appKey: auth.appKey,
      accessToken: auth.accessToken,
    },
  });
  state.ws = socket;

  socket.on("connect", () => {
    state.wsConnected = true;
    socket.emit("subscribe", {
      symbols: [state.symbol],
      wsCapabilityType: "stream-stock-quote",
      preferredProvider: state.provider,
    });
    onReconnectStable();
  });

  socket.on("chart.intraday.point", (payload: any) => {
    const point = payload?.point;
    const cursor = payload?.cursor;
    if (!point?.timestamp) return;
    upsertPoints(state.pointsBySecond, [point]);
    if (typeof cursor === "string" && cursor.length > 0) {
      state.cursor = cursor;
    }
    onSeries(toSortedSeries(state.pointsBySecond));
  });

  socket.on("disconnect", () => {
    state.wsConnected = false;
    onDisconnect();
  });
}

async function disconnectWs(state: WsIntradayState): Promise<void> {
  const socket = state.ws;
  if (!socket) return;

  await new Promise<void>((resolve) => {
    let resolved = false;

    const finish = () => {
      if (resolved) return;
      resolved = true;
      socket.off("unsubscribe-ack", handleAck);
      socket.off("unsubscribe-error", handleError);
      resolve();
    };

    const handleAck = () => finish();
    const handleError = () => finish();

    socket.once("unsubscribe-ack", handleAck);
    socket.once("unsubscribe-error", handleError);

    socket.emit("unsubscribe", {
      symbols: [state.symbol],
      wsCapabilityType: "stream-stock-quote",
    });

    setTimeout(finish, 1000);
  });

  socket.disconnect();
  state.ws = null;
  state.wsConnected = false;
}
```

## 6. 组件内推荐调用顺序

### 6.1 HTTP-only

```ts
await loadSnapshot(state, authHeaders);
render(toSortedSeries(state.pointsBySecond));
startDeltaPolling(state, authHeaders, (series) => render(series));

// 页面卸载或切换 symbol 时
await postJson(
  "/api/v1/chart/intraday-line/release",
  {
    symbol: state.symbol,
    provider: state.provider,
  },
  authHeaders,
);
```

### 6.2 HTTP + WS

```ts
await loadSnapshot(state, authHeaders);
render(toSortedSeries(state.pointsBySecond));

connectWs(
  state,
  { appKey, accessToken },
  (series) => render(series),
  () => startDeltaPolling(state, authHeaders, (series) => render(series)),
  () => stopDeltaPolling(state),
);

// 页面卸载或切换 symbol 时
await disconnectWs(state);
await postJson(
  "/api/v1/chart/intraday-line/release",
  {
    symbol: state.symbol,
    provider: state.provider,
  },
  authHeaders,
);
```

## 7. 错误处理与恢复策略

1. `400 INVALID_ARGUMENT`
- 检查请求体是否传了非白名单字段。
- 当前不要传 `includePrePost`、`since`。

2. `409 CURSOR_EXPIRED`
- 丢弃旧 `cursor`。
- 立即重拉 `snapshot` 并替换全量状态。
- 然后继续 `delta` 轮询或等待 WS 后续推送。

3. `503 PROVIDER_UNAVAILABLE`
- 前端提示“数据源暂不可用”。
- 可保留最近一次图形并展示刷新按钮。

4. 可选前端 WS 断连
- 启动 `delta` `1s` 轮询补洞。
- WS 恢复后可停止高频轮询。

5. `realtimeMergedPoints=0`
- 不要直接判定失败。
- 这通常表示当前响应尚未并入实时点，继续 `delta` 即可。

6. `release`
- 若启用了前端 WS，先执行 `unsubscribe`，再调用 `release`。
- `release` 只负责内部上游订阅，不替代前端 WS 退订。
- 若已释放成功，后续不再调用 `delta` 即可等待服务端停止内部订阅。

## 8. 实现细节建议

1. 统一按“秒桶”做覆盖更新，避免同秒重复点导致图形抖动。
2. `cursor` 视为不透明字符串，不要解码、拼接或篡改。
3. `points` 建议维护为 `Map<second, point>`，渲染前再排序转数组。
4. 当 `supportsFullDay1sHistory=false` 时，不要向用户承诺“完整全日秒级历史回放”。
5. 不要再把“额外手动拉一条 WS 给后端喂数”作为前端接入前提。

## 9. 联调检查清单

1. `snapshot` 返回后是否拿到 `cursor`。
2. 不建立前端 WS 时，`delta` 连续轮询是否仍能持续拿到未来点位。
3. 建立前端 WS 时，是否以 `wsCapabilityType=stream-stock-quote` 订阅成功，并能收到 `chart.intraday.point` 更新图表。
4. `CURSOR_EXPIRED` 时是否能自动重拉 `snapshot` 恢复。
5. 断网或前后台切换后，`delta` 是否能补齐漏点。
6. 页面卸载或切换 symbol 时，前端 WS 是否先收到 `unsubscribe-ack` 或停止继续收点。
7. 随后 `release` 是否成功返回。
8. 是否完全避免“每秒 `snapshot` 全量请求”。
