# 分时折线图前端接入指南

更新时间：2026-03-21

## 1. 适用范围

本文覆盖以下接口和事件：

- `POST /api/v1/chart/intraday-line/snapshot`
- `POST /api/v1/chart/intraday-line/delta`
- `POST /api/v1/chart/intraday-line/release`
- WebSocket 事件：`chart.intraday.point`（主实时增量通道）

当前前端应按以下事实接入：

- 首屏永远来自 `snapshot`
- `snapshot` 响应中的 `sync.realtime` 直接提供后续 WS 订阅所需参数（`wsCapabilityType`、`event`、`preferredProvider`）；非 `live` 模式时为 `null`，表示无需建立 WS
- WebSocket 是主实时增量通道；`delta` 收敛为 WS 断线后的补洞通道
- 当前后端首屏基线仍然是 `1m` 历史，后续增量可能带秒级时间戳
- 前端主图应按分钟桶渲染，不要把每个秒级增量直接追加成新的 x 轴点
- 对外协议已改为”当前用户租约”语义，不再要求前端显式管理 `sessionId`
- `snapshot` 请求里的 `provider` 不是必填；若不显式指定，后端会自动解析，并在 `metadata.provider` 中返回实际使用的 provider
- WS 订阅无需传 `sessionId`；服务端在满足条件时自动按 owner lease 绑定分时图租约

## 2. 接入结论

### 2.1 主图渲染原则

- 主分时图按分钟渲染
- `snapshot` 返回的点可直接归并为分钟桶
- WS `chart.intraday.point` / `delta` 返回的秒级点，统一归并到”当前分钟桶”
- 同一分钟内价格使用最新价覆盖
- 如果前端消费 `volume`，则在同一分钟内累加

### 2.2 参考价原则

- 昨收线使用 `snapshot.reference.previousClosePrice`
- 今开参考价使用 `snapshot.reference.sessionOpenPrice`
- 涨跌额 / 涨跌幅以前收为基准
- 时区展示以 `snapshot.reference.timezone` 为准

### 2.3 租约原则

- `snapshot` 会自动为当前用户建立或续用该标的的活跃租约
- `delta` 会自动续租这份租约
- `release` 只释放当前用户自己的租约
- 不同用户同时看同一 `symbol` 时，彼此 `release` 不会互相影响

## 3. 页面状态设计

推荐维护的状态：

- `cursor`
- `reference`
- `realtimeHint`（来自 `snapshot.sync.realtime`）
- `pointsByMinute`
- `deltaTimer`
- `deltaInFlight`
- `wsConnected`

推荐类型：

```ts
type IntradayPoint = {
  timestamp: string;
  price: number;
  volume: number;
};

type IntradayReference = {
  previousClosePrice: number | null;
  sessionOpenPrice: number | null;
  priceBase: "previous_close";
  marketSession: "regular" | "utc_day";
  timezone: string;
  status: "complete" | "partial" | "unavailable";
};

type IntradaySyncRealtime = {
  wsCapabilityType: string;
  event: string;
  preferredProvider: string;
};

type MinuteLinePoint = {
  timestamp: string;
  price: number;
  volume: number;
};

type IntradayState = {
  symbol: string;
  provider: string | null;
  cursor: string;
  reference: IntradayReference | null;
  realtimeHint: IntradaySyncRealtime | null;
  pointsByMinute: Map<number, MinuteLinePoint>;
  deltaTimer: ReturnType<typeof setTimeout> | null;
  deltaInFlight: boolean;
};
```

## 4. 协议速览

### 4.1 Snapshot

请求：

```json
{
  "symbol": "AAPL.US",
  "pointLimit": 30000
}
```

关键响应字段：

- `data.line.points[]`
- `data.capability.snapshotBaseGranularity`
- `data.capability.supportsFullDay1sHistory`
- `data.reference.previousClosePrice`
- `data.reference.sessionOpenPrice`
- `data.reference.marketSession`
- `data.reference.timezone`
- `data.reference.status`
- `data.sync.cursor`
- `data.sync.lastPointTimestamp`
- `data.sync.serverTime`
- `data.sync.realtime`（live 模式下包含 WS 订阅参数，非 live 为 `null`）
- `data.metadata.provider`

参考响应片段：

```json
{
  "line": {
    "symbol": "AAPL.US",
    "market": "US",
    "tradingDay": "20260313",
    "granularity": "1s",
    "points": [
      {
        "timestamp": "2026-03-13T14:30:00.000Z",
        "price": 255.48,
        "volume": 0
      }
    ]
  },
  "capability": {
    "snapshotBaseGranularity": "1m",
    "supportsFullDay1sHistory": false
  },
  "reference": {
    "previousClosePrice": 255.76,
    "sessionOpenPrice": 255.48,
    "priceBase": "previous_close",
    "marketSession": "regular",
    "timezone": "America/New_York",
    "status": "complete"
  },
  "sync": {
    "cursor": "base64-signed-cursor",
    "lastPointTimestamp": "2026-03-13T14:30:00.000Z",
    "serverTime": "2026-03-13T14:30:01.120Z",
    "realtime": {
      "wsCapabilityType": "stream-stock-quote",
      "event": "chart.intraday.point",
      "preferredProvider": "infoway"
    }
  },
  "metadata": {
    "provider": "infoway"
  }
}
```

说明：

- `line.granularity = “1s”` 不代表主图必须按秒渲染
- 当前真正的首屏基线来源看 `capability.snapshotBaseGranularity`，现在是 `1m`
- `supportsFullDay1sHistory=false` 时，不要向用户承诺”完整全日秒级历史回放”
- `provider` 可选；若前端未显式传入，建议在首个 `snapshot` 后保存 `metadata.provider`，后续 `delta` / `release` / WS `preferredProvider` 优先复用这个已解析值
- 不再需要保存公开 `sessionId`
- `sync.realtime` 在 `metadata.runtimeMode === “live”` 时返回 WS 订阅所需参数（`wsCapabilityType`、`event`、`preferredProvider`），前端据此建立 WS 订阅；非 `live` 模式时为 `null`，表示无需建立 WS

### 4.2 Delta（补洞通道）

请求：

```json
{
  "symbol": "AAPL.US",
  "cursor": "base64-signed-cursor",
  "limit": 2000,
  "strictProviderConsistency": false
}
```

关键响应字段：

- `data.delta.points[]`
- `data.delta.nextCursor`
- `data.delta.hasMore`
- `data.delta.lastPointTimestamp`

说明：

- `delta` 的定位是 WS 断线后的补洞通道，不再作为主轮询通道
- `delta` 的起点 cursor 优先使用”最近一次收到的 WS 事件里的 cursor”，其次使用 `snapshot` 返回的 cursor。注意：WS 事件中的 cursor 不含 `provider` 字段，因此使用 WS cursor 调 `delta` 时应保持 `strictProviderConsistency=false`（默认值），否则会因 provider 缺失返回 `409 CURSOR_EXPIRED`
- 服务端会按”当前用户 + symbol/provider/market”自动续租活跃租约
- 若当前用户不存在对应活跃租约，返回 `409 LEASE_CONFLICT`，需要重新拉 `snapshot`
- 若 `cursor` 失效，返回 `409 CURSOR_EXPIRED`
- 若前端已从 `snapshot.metadata.provider` 拿到实际 provider，推荐后续请求继续携带，避免多 provider 场景下语义漂移

### 4.3 WebSocket（主实时增量通道）

连接地址：

- `/api/v1/stream-receiver/connect`

标准订阅方式（无需 `sessionId`）：

```json
{
  "symbols": ["AAPL.US"],
  "wsCapabilityType": "stream-stock-quote",
  "preferredProvider": "infoway"
}
```

前端应从 `snapshot` 响应的 `sync.realtime` 中获取 `wsCapabilityType` 和 `preferredProvider`，而非硬编码。

当前行为：

- 未携带 `sessionId` 时，若满足分时图标准场景（单 symbol、已认证、请求显式携带 `preferredProvider`），服务端自动按 `ownerIdentity + symbol + provider` 查找 `snapshot` 建立的分时图租约，并将当前 WS client 绑定到该租约
- 自动绑定成功后，应用层 `ping` 会续租当前绑定的分时图租约
- 若 owner lease 未命中（如 `snapshot` 尚未调用、lease 已过期），静默降级为普通 stream 订阅，不报错
- 若查找过程遇到系统异常（Redis 抖动等），同样降级为普通订阅，但以 `warn` 级别记录日志
- 显式携带 `sessionId` 时仍保留遗留兼容路径，但新接入方不应使用

事件载荷（`chart.intraday.point`）：

```json
{
  "symbol": "AAPL.US",
  "market": "US",
  "tradingDay": "20260308",
  "granularity": "1s",
  "point": {
    "timestamp": "2026-03-08T15:42:02.000Z",
    "price": 195.95,
    "volume": 320
  },
  "cursor": "base64-signed-cursor"
}
```

说明：

- `chart.intraday.point` 是事件名，不是 `wsCapabilityType`
- 每次收到 WS 事件后，前端应更新本地 `cursor`，作为后续 `delta` 补洞的起点
- `sessionId` 仅保留为遗留兼容字段，新接入不应再传
- 停止接收推送时，前端应发送 `unsubscribe`
- `unsubscribe` 只负责"前端 Socket → 我们的 WS 服务"，不会替代 `release`
- 若 WS 稳定后暂停高频 `delta`，前端需要继续发送应用层 `ping`；建议 `30s` 一次

### 4.4 Release

请求：

```json
{
  "symbol": "AAPL.US",
  "market": "US"
}
```

响应片段：

```json
{
  "release": {
    "leaseReleased": true,
    "upstreamReleased": false,
    "reason": "RELEASED",
    "symbol": "AAPL.US",
    "market": "US",
    "provider": "infoway",
    "wsCapabilityType": "stream-stock-quote",
    "activeLeaseCount": 0,
    "graceExpiresAt": "2026-03-20T11:51:54.702Z"
  }
}
```

说明：

- `release` 只释放当前用户的租约
- `provider` 可选；省略时，服务端会尝试按“当前用户 + symbol + market”释放唯一活跃租约
- 若省略 `provider` 且当前用户在同一 `symbol` 上存在多个 provider 的活跃租约，服务端会返回 `409`
- `release` 不负责替当前前端 WebSocket 连接执行 `unsubscribe`
- 重复释放按幂等成功处理
- 若命中 `ALREADY_RELEASED`，`provider` 可能为空字符串，`graceExpiresAt` 也可能为 `null` 或时间戳；前端不应把这两个字段当成固定值
- 如果前端启用了 WS，推荐顺序是先 `unsubscribe`，再调用 `release`

## 5. 推荐接入模式

### 5.1 HTTP-only（降级模式）

1. 页面加载后调用一次 `snapshot`
2. 拿到 `points`、`reference`、`cursor` 后立即渲染首屏
3. 启动 `delta` 轮询，持续消费未来点位
4. 若收到 `CURSOR_EXPIRED` 或 `LEASE_CONFLICT`，立即重拉 `snapshot`
5. 页面卸载或切换 `symbol` 时调用 `release`
6. 任意时刻禁止每秒重拉 `snapshot`

### 5.2 HTTP + WS（标准推荐）

1. 页面加载后调用一次 `snapshot`
2. 保存 `cursor`、`reference` 并渲染首屏
3. 从 `snapshot` 响应的 `sync.realtime` 获取 `wsCapabilityType` 和 `preferredProvider`
4. 建立前端 WS，使用 `subscribe({ symbols: [symbol], wsCapabilityType, preferredProvider })` 订阅（无需传 `sessionId`）
5. 服务端自动按 owner lease 绑定 WS client 到分时图租约，`ping` 自动续租
6. WS 作为主增量通道持续接收 `chart.intraday.point`，每次更新本地 `cursor`
7. WS 断线后重连，用最近 `cursor` 调 `delta` 补洞，补齐后继续消费 WS
8. 页面卸载或切换 `symbol` 时，先发送 `unsubscribe`，再调用 `release`

## 6. 分钟桶归并规则

建议实现：

```ts
function toMinuteBucket(ts: string): number {
  const date = new Date(ts);
  date.setUTCSeconds(0, 0);
  return date.getTime();
}

function mergePointIntoMinuteBucket(
  pointsByMinute: Map<number, MinuteLinePoint>,
  point: IntradayPoint,
): void {
  const bucket = toMinuteBucket(point.timestamp);
  const existing = pointsByMinute.get(bucket);

  if (!existing) {
    pointsByMinute.set(bucket, {
      timestamp: new Date(bucket).toISOString(),
      price: point.price,
      volume: point.volume ?? 0,
    });
    return;
  }

  pointsByMinute.set(bucket, {
    timestamp: existing.timestamp,
    price: point.price,
    volume: (existing.volume ?? 0) + (point.volume ?? 0),
  });
}

function mergePoints(
  pointsByMinute: Map<number, MinuteLinePoint>,
  incoming: IntradayPoint[],
): void {
  for (const point of incoming) {
    mergePointIntoMinuteBucket(pointsByMinute, point);
  }
}

function toSortedSeries(
  pointsByMinute: Map<number, MinuteLinePoint>,
): MinuteLinePoint[] {
  return Array.from(pointsByMinute.entries())
    .sort((a, b) => a[0] - b[0])
    .map(([, point]) => point);
}
```

## 7. 前端实现模板（TypeScript）

### 7.1 HTTP-only

```ts
type SnapshotResponse = {
  line: {
    points: IntradayPoint[];
  };
  reference: IntradayReference;
  metadata: {
    provider: string;
    runtimeMode: string;
  };
  sync: {
    cursor: string;
    realtime: IntradaySyncRealtime | null;
  };
};

type DeltaResponse = {
  delta: {
    points: IntradayPoint[];
    hasMore: boolean;
    nextCursor: string;
  };
};

const API_BASE_URL = "http://127.0.0.1:3001";

async function postJson<T>(
  path: string,
  body: Record<string, unknown>,
  headers: Record<string, string>,
): Promise<T> {
  const resp = await fetch(`${API_BASE_URL}${path}`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      ...headers,
    },
    body: JSON.stringify(body),
  });

  const json = await resp.json();
  if (!resp.ok || json?.success !== true) {
    const err = new Error(json?.message || `HTTP ${resp.status}`);
    (err as any).payload = json;
    throw err;
  }

  return json.data as T;
}

async function loadSnapshot(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  const data = await postJson<SnapshotResponse>(
    "/api/v1/chart/intraday-line/snapshot",
    {
      symbol: state.symbol,
      pointLimit: 30000,
    },
    authHeaders,
  );

  state.pointsByMinute.clear();
  mergePoints(state.pointsByMinute, data.line.points || []);
  state.reference = data.reference;
  state.provider = data.metadata?.provider || state.provider || null;
  state.cursor = data.sync.cursor;
  state.realtimeHint = data.sync.realtime || null;
}

async function pullDeltaOnce(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  const body: Record<string, unknown> = {
    symbol: state.symbol,
    cursor: state.cursor,
    limit: 2000,
    strictProviderConsistency: false,
  };
  if (state.provider) {
    body.provider = state.provider;
  }

  const data = await postJson<DeltaResponse>(
    "/api/v1/chart/intraday-line/delta",
    body,
    authHeaders,
  );

  mergePoints(state.pointsByMinute, data.delta.points || []);
  state.cursor = data.delta.nextCursor;
}

async function releaseLease(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  const body: Record<string, unknown> = {
    symbol: state.symbol,
  };
  if (state.provider) {
    body.provider = state.provider;
  }

  await postJson(
    "/api/v1/chart/intraday-line/release",
    body,
    authHeaders,
  );
}

function stopDeltaPolling(state: IntradayState): void {
  if (!state.deltaTimer) return;
  clearTimeout(state.deltaTimer);
  state.deltaTimer = null;
}

function startDeltaPolling(
  state: IntradayState,
  authHeaders: Record<string, string>,
  onSeries: (series: MinuteLinePoint[]) => void,
  intervalMs: number = 1000,
): void {
  if (state.deltaTimer) return;

  const tick = async () => {
    if (state.deltaInFlight) {
      state.deltaTimer = setTimeout(tick, intervalMs);
      return;
    }

    state.deltaInFlight = true;
    try {
      await pullDeltaOnce(state, authHeaders);
      onSeries(toSortedSeries(state.pointsByMinute));
    } catch (error: any) {
      const message = String(error?.payload?.message || error?.message || "");
      if (message.includes("CURSOR_EXPIRED") || message.includes("LEASE_CONFLICT")) {
        await loadSnapshot(state, authHeaders);
        onSeries(toSortedSeries(state.pointsByMinute));
      }
    } finally {
      state.deltaInFlight = false;
      state.deltaTimer = setTimeout(tick, intervalMs);
    }
  };

  state.deltaTimer = setTimeout(tick, intervalMs);
}
```

### 7.2 WS 标准接入

```ts
import { io, Socket } from "socket.io-client";

type WsIntradayState = IntradayState & {
  ws: Socket | null;
  wsConnected: boolean;
  wsPingTimer: ReturnType<typeof setInterval> | null;
};

function connectWs(
  state: WsIntradayState,
  auth: { appKey: string; accessToken: string },
  onSeries: (series: MinuteLinePoint[]) => void,
  onDisconnect: () => void,
  onReconnectStable: () => void,
): void {
  // sync.realtime 为 null 时（非 live 模式），不应建立 WS
  if (!state.realtimeHint) return;

  const socket = io(API_BASE_URL, {
    path: "/api/v1/stream-receiver/connect",
    transports: ["websocket"],
    auth: {
      appKey: auth.appKey,
      accessToken: auth.accessToken,
    },
  });

  state.ws = socket;

  socket.on("connect", () => {
    state.wsConnected = true;
    // 使用 snapshot.sync.realtime 提供的参数，无需传 sessionId
    socket.emit("subscribe", {
      symbols: [state.symbol],
      wsCapabilityType: state.realtimeHint!.wsCapabilityType,
      preferredProvider: state.realtimeHint!.preferredProvider,
    });
    state.wsPingTimer = setInterval(() => {
      socket.emit("ping");
    }, 30_000);
    onReconnectStable();
  });

  socket.on("chart.intraday.point", (payload: any) => {
    const point = payload?.point;
    const cursor = payload?.cursor;
    if (!point?.timestamp) return;

    mergePoints(state.pointsByMinute, [point]);

    if (typeof cursor === "string" && cursor.length > 0) {
      state.cursor = cursor;
    }

    onSeries(toSortedSeries(state.pointsByMinute));
  });

  socket.on("disconnect", () => {
    state.wsConnected = false;
    onDisconnect();
  });
}

async function disconnectWs(state: WsIntradayState): Promise<void> {
  const socket = state.ws;
  if (!socket) return;

  await new Promise<void>((resolve) => {
    let resolved = false;

    const finish = () => {
      if (resolved) return;
      resolved = true;
      socket.off("unsubscribe-ack", handleAck);
      socket.off("unsubscribe-error", handleError);
      resolve();
    };

    const handleAck = () => finish();
    const handleError = () => finish();

    socket.once("unsubscribe-ack", handleAck);
    socket.once("unsubscribe-error", handleError);

    socket.emit("unsubscribe", {
      symbols: [state.symbol],
      wsCapabilityType: state.realtimeHint?.wsCapabilityType || "stream-stock-quote",
    });

    setTimeout(finish, 1000);
  });

  socket.disconnect();
  if (state.wsPingTimer) {
    clearInterval(state.wsPingTimer);
    state.wsPingTimer = null;
  }
  state.ws = null;
  state.wsConnected = false;
}
```

## 8. 错误处理与恢复策略

1. `400 INVALID_ARGUMENT`

- 检查请求体是否传了非法字段
- 当前不要传 `since`

2. `409 CURSOR_EXPIRED`

- 丢弃旧 `cursor`
- 立即重拉 `snapshot`

3. `409 LEASE_CONFLICT`

- 当前用户在该标的上已无活跃租约
- 丢弃旧 `cursor`
- 立即重拉 `snapshot`

4. `503 PROVIDER_UNAVAILABLE`

- 提示“数据源暂不可用”
- 可保留最近一次图形并展示刷新按钮

5. WS 断连

- 用最近一次 WS 事件中的 `cursor` 调 `delta` 补洞
- 注意：WS cursor 不含 `provider` 字段，调 `delta` 时应保持 `strictProviderConsistency=false`
- WS 恢复后停止 `delta` 轮询，继续消费 WS 主通道

6. `reference.status = partial | unavailable`

- 不要自行补算昨收或今开
- 能展示哪个值就展示哪个值

7. `release`

- 若启用了前端 WS，先执行 `unsubscribe`，再调用 `release`
- `release` 负责释放当前用户租约，并按共享引用情况决定是否真实上游退订；不替代前端 WS 退订
- 若返回 `leaseReleased=false` 且 `reason=ALREADY_RELEASED`，按清理成功处理即可


## 额外补充
  WS unsubscribe

  - 作用范围是“前端这个 WebSocket 连接”和“我们自己的 WS 服务”之间。
  - 它的意思是：这个 socket 不要再给我推这个 symbol 的 chart.intraday.point 了。
  - 它解决的是“停止接收实时消息”。
  - 它不会释放分时图租约，也不会替你清理后端为 snapshot/delta 建立的 owner lease。

  API release

  - 作用范围是“当前调用方 owner”在后端持有的分时图租约。
  - 它的意思是：这个用户/调用方不再消费这个 symbol 的分时图了，后端可以释放这份 lease。
  - 它解决的是“结束后端资源占用、结束后续 delta 能力”。
  - 调完之后，再用旧 cursor 调 delta，会得到 409 LEASE_CONFLICT。

  说白话：

  - unsubscribe = “别再往我这个前端连接上推消息了”
  - release = “我这个用户彻底不用这个分时图了，把后端租约收掉”

  为什么两个都要有：

  - 只 unsubscribe 不 release
    前端不收消息了，但后端租约还在，占着资源，之后 delta 可能还能继续用。
  - 只 release 不 unsubscribe
    后端 lease 释放了，但这个 WS 连接本身可能还挂着订阅状态，语义不干净，也不符合推荐收尾顺序。

  当前正确顺序：

  - 页面关闭/切 symbol 时，先 WS unsubscribe
  - 再调用 POST /api/v1/chart/intraday-line/release

## 9. 联调检查清单

1. `snapshot` 返回后是否拿到 `cursor`、`reference`
2. `snapshot` 返回的 `sync.realtime` 是否在 live 模式下包含 `wsCapabilityType`、`event`、`preferredProvider`；非 live 模式下是否为 `null`
3. `reference.previousClosePrice`、`reference.sessionOpenPrice` 是否可直接用于 UI
4. 不建立前端 WS 时，`delta` 是否能持续拿到未来点位
5. 建立前端 WS 时，是否使用 `sync.realtime` 提供的参数订阅（无需传 `sessionId`），并收到 `chart.intraday.point`
6. WS 事件中的 `cursor` 是否被正确保存，用于后续 `delta` 补洞
7. 秒级增量是否被正确归并到当前分钟桶
8. `CURSOR_EXPIRED` / `LEASE_CONFLICT` 时是否能自动重拉 `snapshot`
9. 页面卸载或切换 `symbol` 时，若启用了 WS，是否先 `unsubscribe`，再 `release`
10. 是否完全避免”每秒 `snapshot` 全量请求”
