# 分时折线图前端接入指南

更新时间：2026-03-17

## 1. 适用范围

本文覆盖以下接口和事件：

- `POST /api/v1/chart/intraday-line/snapshot`
- `POST /api/v1/chart/intraday-line/delta`
- `POST /api/v1/chart/intraday-line/release`
- 可选 WebSocket 事件：`chart.intraday.point`

当前前端应按以下事实接入：

- 首屏永远来自 `snapshot`
- 后续更新来自 `delta` 和可选 `WS`
- 当前后端首屏基线仍然是 `1m` 历史，后续增量可能带秒级时间戳
- 前端主图应按分钟桶渲染，不要把每个秒级增量直接追加成新的 x 轴点
- 对外协议已改为“当前用户租约”语义，不再要求前端显式管理 `sessionId`

## 2. 接入结论

### 2.1 主图渲染原则

- 主分时图按分钟渲染
- `snapshot` 返回的点可直接归并为分钟桶
- `delta` / `WS` 返回的秒级点，统一归并到“当前分钟桶”
- 同一分钟内价格使用最新价覆盖
- 如果前端消费 `volume`，则在同一分钟内累加

### 2.2 参考价原则

- 昨收线使用 `snapshot.reference.previousClosePrice`
- 今开参考价使用 `snapshot.reference.sessionOpenPrice`
- 涨跌额 / 涨跌幅以前收为基准
- 时区展示以 `snapshot.reference.timezone` 为准

### 2.3 租约原则

- `snapshot` 会自动为当前用户建立或续用该标的的活跃租约
- `delta` 会自动续租这份租约
- `release` 只释放当前用户自己的租约
- 不同用户同时看同一 `symbol` 时，彼此 `release` 不会互相影响

## 3. 页面状态设计

推荐维护的状态：

- `cursor`
- `reference`
- `pointsByMinute`
- `deltaTimer`
- `deltaInFlight`
- `wsConnected`（启用前端 WS 时）

推荐类型：

```ts
type IntradayPoint = {
  timestamp: string;
  price: number;
  volume: number;
};

type IntradayReference = {
  previousClosePrice: number | null;
  sessionOpenPrice: number | null;
  priceBase: "previous_close";
  marketSession: "regular" | "utc_day";
  timezone: string;
  status: "complete" | "partial" | "unavailable";
};

type MinuteLinePoint = {
  timestamp: string;
  price: number;
  volume: number;
};

type IntradayState = {
  symbol: string;
  provider: string;
  cursor: string;
  reference: IntradayReference | null;
  pointsByMinute: Map<number, MinuteLinePoint>;
  deltaTimer: ReturnType<typeof setTimeout> | null;
  deltaInFlight: boolean;
};
```

## 4. 协议速览

### 4.1 Snapshot

请求：

```json
{
  "symbol": "AAPL.US",
  "provider": "infoway",
  "pointLimit": 30000
}
```

关键响应字段：

- `data.line.points[]`
- `data.capability.snapshotBaseGranularity`
- `data.capability.supportsFullDay1sHistory`
- `data.reference.previousClosePrice`
- `data.reference.sessionOpenPrice`
- `data.reference.marketSession`
- `data.reference.timezone`
- `data.reference.status`
- `data.sync.cursor`
- `data.sync.lastPointTimestamp`
- `data.sync.serverTime`

参考响应片段：

```json
{
  "line": {
    "symbol": "AAPL.US",
    "market": "US",
    "tradingDay": "20260313",
    "granularity": "1s",
    "points": [
      {
        "timestamp": "2026-03-13T14:30:00.000Z",
        "price": 255.48,
        "volume": 0
      }
    ]
  },
  "capability": {
    "snapshotBaseGranularity": "1m",
    "supportsFullDay1sHistory": false
  },
  "reference": {
    "previousClosePrice": 255.76,
    "sessionOpenPrice": 255.48,
    "priceBase": "previous_close",
    "marketSession": "regular",
    "timezone": "America/New_York",
    "status": "complete"
  },
  "sync": {
    "cursor": "base64-signed-cursor",
    "lastPointTimestamp": "2026-03-13T14:30:00.000Z",
    "serverTime": "2026-03-13T14:30:01.120Z"
  }
}
```

说明：

- `line.granularity = "1s"` 不代表主图必须按秒渲染
- 当前真正的首屏基线来源看 `capability.snapshotBaseGranularity`，现在是 `1m`
- `supportsFullDay1sHistory=false` 时，不要向用户承诺“完整全日秒级历史回放”
- 不再需要保存公开 `sessionId`

### 4.2 Delta

请求：

```json
{
  "symbol": "AAPL.US",
  "provider": "infoway",
  "cursor": "base64-signed-cursor",
  "limit": 2000,
  "strictProviderConsistency": false
}
```

关键响应字段：

- `data.delta.points[]`
- `data.delta.nextCursor`
- `data.delta.hasMore`
- `data.delta.lastPointTimestamp`

说明：

- `delta` 只依赖 `cursor`
- 服务端会按“当前用户 + symbol/provider/market”自动续租活跃租约
- 若当前用户不存在对应活跃租约，返回 `409 LEASE_CONFLICT`，需要重新拉 `snapshot`
- 若 `cursor` 失效，返回 `409 CURSOR_EXPIRED`

### 4.3 WebSocket（可选增强）

连接地址：

- `/api/v1/stream-receiver/connect`

推荐订阅参数：

```json
{
  "symbols": ["AAPL.US"],
  "wsCapabilityType": "stream-stock-quote",
  "preferredProvider": "infoway"
}
```

当前行为：

- 当只订阅一个 `symbol` 且当前用户已持有该标的的活跃租约时，服务端会自动匹配并绑定该租约
- 若显式传入 `preferredProvider`，会按该 provider 精确匹配租约
- 若未传 `preferredProvider`，当前实现会按“当前用户 + symbol”查找；若命中多个 provider 的活跃租约，则按最近活跃租约回退
- 因此推荐 `preferredProvider` 与最近一次 `snapshot` 使用的 provider 保持一致
- 服务端会广播统一后的分时领域事件 `chart.intraday.point`
- 停止接收推送时，前端应发送 `unsubscribe`
- 若 WS 稳定后暂停高频 `delta`，前端需要继续发送应用层 `ping`；建议 `30s` 一次

说明：

- `chart.intraday.point` 是事件名，不是 `wsCapabilityType`
- `sessionId` 仅保留为遗留兼容字段，新接入不应再传
- 即使启用 WS，`delta` 仍建议保留为补洞和恢复接口

### 4.4 Release

请求：

```json
{
  "symbol": "AAPL.US",
  "provider": "infoway",
  "market": "US"
}
```

响应片段：

```json
{
  "release": {
    "leaseReleased": false,
    "upstreamReleased": false,
    "reason": "ALREADY_RELEASED",
    "symbol": "AAPL.US",
    "market": "US",
    "provider": "infoway",
    "wsCapabilityType": "stream-stock-quote",
    "activeLeaseCount": 0,
    "graceExpiresAt": "2026-03-17T09:31:00.000Z"
  }
}
```

说明：

- `release` 只释放当前用户的租约
- `release` 不负责替当前前端 WebSocket 连接执行 `unsubscribe`
- 重复释放按幂等成功处理
- 如果前端启用了 WS，推荐顺序是先 `unsubscribe`，再调用 `release`

## 5. 推荐接入模式

### 5.1 HTTP-only

1. 页面加载后调用一次 `snapshot`
2. 拿到 `points`、`reference`、`cursor` 后立即渲染首屏
3. 启动 `delta` 轮询，持续消费未来点位
4. 若收到 `CURSOR_EXPIRED` 或 `LEASE_CONFLICT`，立即重拉 `snapshot`
5. 页面卸载或切换 `symbol` 时调用 `release`
6. 任意时刻禁止每秒重拉 `snapshot`

### 5.2 HTTP + WS

1. 页面加载后调用一次 `snapshot`
2. 保存 `cursor` 并渲染首屏
3. 建立前端 WS，发送单 symbol `subscribe`
4. WS 正常时，可暂停高频 `delta`，但继续发送应用层 `ping`
5. WS 断开时，立即启用 `delta` `1s` 轮询补洞
6. WS 恢复稳定后，可停止高频 `delta`
7. 页面卸载或切换 `symbol` 时，先发送 `unsubscribe`
8. 随后调用 `release`

## 6. 分钟桶归并规则

建议实现：

```ts
function toMinuteBucket(ts: string): number {
  const date = new Date(ts);
  date.setUTCSeconds(0, 0);
  return date.getTime();
}

function mergePointIntoMinuteBucket(
  pointsByMinute: Map<number, MinuteLinePoint>,
  point: IntradayPoint,
): void {
  const bucket = toMinuteBucket(point.timestamp);
  const existing = pointsByMinute.get(bucket);

  if (!existing) {
    pointsByMinute.set(bucket, {
      timestamp: new Date(bucket).toISOString(),
      price: point.price,
      volume: point.volume ?? 0,
    });
    return;
  }

  pointsByMinute.set(bucket, {
    timestamp: existing.timestamp,
    price: point.price,
    volume: (existing.volume ?? 0) + (point.volume ?? 0),
  });
}

function mergePoints(
  pointsByMinute: Map<number, MinuteLinePoint>,
  incoming: IntradayPoint[],
): void {
  for (const point of incoming) {
    mergePointIntoMinuteBucket(pointsByMinute, point);
  }
}

function toSortedSeries(
  pointsByMinute: Map<number, MinuteLinePoint>,
): MinuteLinePoint[] {
  return Array.from(pointsByMinute.entries())
    .sort((a, b) => a[0] - b[0])
    .map(([, point]) => point);
}
```

## 7. 前端实现模板（TypeScript）

### 7.1 HTTP-only

```ts
type SnapshotResponse = {
  line: {
    points: IntradayPoint[];
  };
  reference: IntradayReference;
  sync: {
    cursor: string;
  };
};

type DeltaResponse = {
  delta: {
    points: IntradayPoint[];
    hasMore: boolean;
    nextCursor: string;
  };
};

const API_BASE_URL = "http://127.0.0.1:3001";

async function postJson<T>(
  path: string,
  body: Record<string, unknown>,
  headers: Record<string, string>,
): Promise<T> {
  const resp = await fetch(`${API_BASE_URL}${path}`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      ...headers,
    },
    body: JSON.stringify(body),
  });

  const json = await resp.json();
  if (!resp.ok || json?.success !== true) {
    const err = new Error(json?.message || `HTTP ${resp.status}`);
    (err as any).payload = json;
    throw err;
  }

  return json.data as T;
}

async function loadSnapshot(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  const data = await postJson<SnapshotResponse>(
    "/api/v1/chart/intraday-line/snapshot",
    {
      symbol: state.symbol,
      provider: state.provider,
      pointLimit: 30000,
    },
    authHeaders,
  );

  state.pointsByMinute.clear();
  mergePoints(state.pointsByMinute, data.line.points || []);
  state.reference = data.reference;
  state.cursor = data.sync.cursor;
}

async function pullDeltaOnce(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  const data = await postJson<DeltaResponse>(
    "/api/v1/chart/intraday-line/delta",
    {
      symbol: state.symbol,
      provider: state.provider,
      cursor: state.cursor,
      limit: 2000,
      strictProviderConsistency: false,
    },
    authHeaders,
  );

  mergePoints(state.pointsByMinute, data.delta.points || []);
  state.cursor = data.delta.nextCursor;
}

async function releaseLease(
  state: IntradayState,
  authHeaders: Record<string, string>,
): Promise<void> {
  await postJson(
    "/api/v1/chart/intraday-line/release",
    {
      symbol: state.symbol,
      provider: state.provider,
    },
    authHeaders,
  );
}

function stopDeltaPolling(state: IntradayState): void {
  if (!state.deltaTimer) return;
  clearTimeout(state.deltaTimer);
  state.deltaTimer = null;
}

function startDeltaPolling(
  state: IntradayState,
  authHeaders: Record<string, string>,
  onSeries: (series: MinuteLinePoint[]) => void,
  intervalMs: number = 1000,
): void {
  if (state.deltaTimer) return;

  const tick = async () => {
    if (state.deltaInFlight) {
      state.deltaTimer = setTimeout(tick, intervalMs);
      return;
    }

    state.deltaInFlight = true;
    try {
      await pullDeltaOnce(state, authHeaders);
      onSeries(toSortedSeries(state.pointsByMinute));
    } catch (error: any) {
      const message = String(error?.payload?.message || error?.message || "");
      if (message.includes("CURSOR_EXPIRED") || message.includes("LEASE_CONFLICT")) {
        await loadSnapshot(state, authHeaders);
        onSeries(toSortedSeries(state.pointsByMinute));
      }
    } finally {
      state.deltaInFlight = false;
      state.deltaTimer = setTimeout(tick, intervalMs);
    }
  };

  state.deltaTimer = setTimeout(tick, intervalMs);
}
```

### 7.2 可选 WS 增强

```ts
import { io, Socket } from "socket.io-client";

type WsIntradayState = IntradayState & {
  ws: Socket | null;
  wsConnected: boolean;
  wsPingTimer: ReturnType<typeof setInterval> | null;
};

function connectWs(
  state: WsIntradayState,
  auth: { appKey: string; accessToken: string },
  onSeries: (series: MinuteLinePoint[]) => void,
  onDisconnect: () => void,
  onReconnectStable: () => void,
): void {
  const socket = io(API_BASE_URL, {
    path: "/api/v1/stream-receiver/connect",
    transports: ["websocket"],
    auth: {
      appKey: auth.appKey,
      accessToken: auth.accessToken,
    },
  });

  state.ws = socket;

  socket.on("connect", () => {
    state.wsConnected = true;
    socket.emit("subscribe", {
      symbols: [state.symbol],
      wsCapabilityType: "stream-stock-quote",
      preferredProvider: state.provider,
    });
    state.wsPingTimer = setInterval(() => {
      socket.emit("ping");
    }, 30_000);
    onReconnectStable();
  });

  socket.on("chart.intraday.point", (payload: any) => {
    const point = payload?.point;
    const cursor = payload?.cursor;
    if (!point?.timestamp) return;

    mergePoints(state.pointsByMinute, [point]);

    if (typeof cursor === "string" && cursor.length > 0) {
      state.cursor = cursor;
    }

    onSeries(toSortedSeries(state.pointsByMinute));
  });

  socket.on("disconnect", () => {
    state.wsConnected = false;
    onDisconnect();
  });
}

async function disconnectWs(state: WsIntradayState): Promise<void> {
  const socket = state.ws;
  if (!socket) return;

  await new Promise<void>((resolve) => {
    let resolved = false;

    const finish = () => {
      if (resolved) return;
      resolved = true;
      socket.off("unsubscribe-ack", handleAck);
      socket.off("unsubscribe-error", handleError);
      resolve();
    };

    const handleAck = () => finish();
    const handleError = () => finish();

    socket.once("unsubscribe-ack", handleAck);
    socket.once("unsubscribe-error", handleError);

    socket.emit("unsubscribe", {
      symbols: [state.symbol],
      wsCapabilityType: "stream-stock-quote",
    });

    setTimeout(finish, 1000);
  });

  socket.disconnect();
  if (state.wsPingTimer) {
    clearInterval(state.wsPingTimer);
    state.wsPingTimer = null;
  }
  state.ws = null;
  state.wsConnected = false;
}
```

## 8. 错误处理与恢复策略

1. `400 INVALID_ARGUMENT`

- 检查请求体是否传了非法字段
- 当前不要传 `since`

2. `409 CURSOR_EXPIRED`

- 丢弃旧 `cursor`
- 立即重拉 `snapshot`

3. `409 LEASE_CONFLICT`

- 当前用户在该标的上已无活跃租约
- 丢弃旧 `cursor`
- 立即重拉 `snapshot`

4. `503 PROVIDER_UNAVAILABLE`

- 提示“数据源暂不可用”
- 可保留最近一次图形并展示刷新按钮

5. WS 断连

- 启动 `delta` `1s` 轮询补洞
- WS 恢复后可停止高频轮询

6. `reference.status = partial | unavailable`

- 不要自行补算昨收或今开
- 能展示哪个值就展示哪个值

7. `release`

- 若启用了前端 WS，先执行 `unsubscribe`，再调用 `release`
- `release` 只负责内部上游订阅，不替代前端 WS 退订
- 若返回 `leaseReleased=false` 且 `reason=ALREADY_RELEASED`，按清理成功处理即可

## 9. 联调检查清单

1. `snapshot` 返回后是否拿到 `cursor`、`reference`
2. `reference.previousClosePrice`、`reference.sessionOpenPrice` 是否可直接用于 UI
3. 不建立前端 WS 时，`delta` 是否能持续拿到未来点位
4. 建立前端 WS 时，是否用单 symbol + `preferredProvider` 订阅成功，并收到 `chart.intraday.point`
5. 秒级增量是否被正确归并到当前分钟桶
6. `CURSOR_EXPIRED` / `LEASE_CONFLICT` 时是否能自动重拉 `snapshot`
7. 页面卸载或切换 `symbol` 时，若启用了 WS，是否先 `unsubscribe`，再 `release`
8. 是否完全避免“每秒 `snapshot` 全量请求”
