# 其他应用开发者对接指南

更新时间：2026-03-08
适用对象：需要把 `newstockapi` 作为行情中台接入自己应用（Web、移动端、桌面端、后端服务）的开发者。

## 1. 接入架构与能力边界

`newstockapi` 提供三类接入面：

- 强时效查询：`POST /api/v1/receiver/data`
- 弱时效查询：`POST /api/v1/query/execute`
- 实时订阅：WebSocket `GET /api/v1/stream-receiver/connect`

推荐场景：

- 行情卡片/详情页首屏：`receiver/data`
- 历史分时：`receiver/data + get-stock-history`
- 实时行情：`stream-receiver/connect + subscribe`

## 2. 前置条件

### 2.1 服务端环境变量（最小集）

- 通用：`PORT`、`MONGODB_URI`、`REDIS_URL`、`JWT_SECRET`
- JvQuant：`JVQUANT_TOKEN`
- Infoway：`INFOWAY_API_KEY`

建议同时显式配置 `.env` 中的默认值项（已在仓库模板补齐），避免不同环境的隐式默认差异。

### 2.2 鉴权凭证

推荐使用 API Key 双头模式：

- `X-App-Key: <appKey>`
- `X-Access-Token: <accessToken>`

最小只读权限建议：

- `data:read`
- `query:execute`
- `stream:read`
- `stream:subscribe`
- `providers:read`

## 3. 快速开始（REST）

### 3.1 获取最新行情

请求：`POST /api/v1/receiver/data`

```json
{
  "symbols": ["AAPL.US"],
  "receiverType": "get-stock-quote",
  "options": {
    "preferredProvider": "infoway",
    "market": "US"
  }
}
```

### 3.2 获取分时历史（用于首屏回填）

请求：`POST /api/v1/receiver/data`

```json
{
  "symbols": ["AAPL.US"],
  "receiverType": "get-stock-history",
  "options": {
    "preferredProvider": "infoway",
    "market": "US"
  }
}
```

客户端合并规则：

- 先用历史数据初始化序列。
- 再接入实时推送。
- 以 `symbol + minute(timestamp)` 作为幂等键 upsert。

## 4. 快速开始（WebSocket）

连接：`/api/v1/stream-receiver/connect`（仅 `websocket`）

订阅示例：

```json
{
  "symbols": ["AAPL.US", "00700.HK"],
  "wsCapabilityType": "stream-stock-quote",
  "preferredProvider": "infoway"
}
```

关键事件：

- 入站：`subscribe`、`unsubscribe`、`ping`、`request-recovery`
- 出站：`subscribe-ack`、`data`、`pong`、`recovery-data`

## 5. Provider 选择策略

### 5.1 显式指定（推荐）

在请求 `options.preferredProvider` 或订阅 `preferredProvider` 中指定：

- `infoway`
- `jvquant`
- `longport`

### 5.2 默认优先级

默认按 `.env` 的 `PROVIDER_PRIORITY_*` 规则选择。若你对稳定性有固定要求，请在应用侧显式指定 provider，不依赖默认抢占。

## 6. 恢复与容错

断线重连后建议立即发送 `request-recovery`，并开启服务端历史兜底配置：

- `RECOVERY_HISTORY_FALLBACK_ENABLED=true`
- `RECOVERY_GAP_THRESHOLD_MS=90000`
- `RECOVERY_MIN_COVERAGE_RATIO=0.6`
- `RECOVERY_MAX_HISTORY_POINTS=240`

当缓存覆盖不足时，服务端会走历史接口补齐，降低分时图断档概率。

## 7. 常见问题与排障

### 7.1 401/403 鉴权失败

- 检查 `X-App-Key`、`X-Access-Token` 是否同时传递。
- 检查 API Key 权限是否包含 `stream:subscribe` 等读权限。

### 7.2 订阅成功但无数据

- 先确认 symbol 是否使用标准格式：`*.HK/*.US/*.SH/*.SZ`。
- 确认市场是否处于交易时段。
- 检查 `preferredProvider` 在该能力上是否真实支持。

### 7.3 历史分时点位偏少

- 检查 `INFOWAY_INTRADAY_KLINE_NUM` 和 `RECOVERY_MAX_HISTORY_POINTS`。
- `get-stock-history` 当前是 MVP 单标的，批量符号需拆请求。

## 8. 对接联调建议

- 先跑 REST 烟测，再接 WebSocket。
- 先固定单 provider，联调通过后再启用 provider 优先级策略。
- 联调期记录 `requestId/provider/timestamp`，方便跨系统对账。
