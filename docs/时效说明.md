# 时效说明（入口与更新机制）

更新时间：2026-03-10 22:50:39 CST  
口径说明：本说明以当前代码实现为准，不以 Swagger 描述文案为准。

## 1. 统一响应格式（先看这个）

所有 HTTP 入口都走全局 `ResponseInterceptor`，统一包装为：

```json
{
  "success": true,
  "statusCode": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": "2026-03-10T00:00:00.000Z"
}
```

实现位置：
- `src/main.ts`（全局注册拦截器）
- `src/common/core/interceptors/response.interceptor.ts`

## 2. 入口总览（路径 + 时效）

| 入口 | 路径 | 时效定位 | 默认策略/TTL | 更新触发机制 |
|---|---|---|---|---|
| 强时效数据接口 | `POST /api/v1/receiver/data` | 强时效 | `STRONG_TIMELINESS`，默认 `5s`（`SMART_CACHE_TTL_STRONG_S`） | 读穿透：命中直接返回；未命中/过期回源并写缓存 |
| 弱时效查询接口 | `POST /api/v1/query/execute` | 弱时效 | `WEAK_TIMELINESS`，默认 `300s`（`SMART_CACHE_TTL_WEAK_S`） | 读穿透：命中直接返回；未命中/过期回源并写缓存 |
| support-list 元数据 | `GET /api/v1/query/support-list/meta` | 弱时效（版本同步） | 非 SmartCache TTL 模式 | 读取前先 `ensureTypeInitialized`，必要时触发刷新 |
| support-list 全量/增量 | `GET /api/v1/query/support-list` | 弱时效（版本同步） | 增量保留窗口 `7天` | 按版本链返回全量/增量，链断裂返回 `409` 要求重同步 |
| 分时线快照 | `POST /api/v1/chart/intraday-line/snapshot` | 强时效读模型 | 非通用 TTL 模式 | `1m` 历史基线 + 实时 `1s` 增量合并 |
| 分时线增量 | `POST /api/v1/chart/intraday-line/delta` | 强时效增量 | 非通用 TTL 模式 | 基于 `cursor` 继续拉取增量点 |
| 实时流 | `WS /api/v1/stream-receiver/connect` | 实时推送 | Hot `5000ms`，Warm `300s` | 订阅后持续推送，流缓存分热/温层 |

## 3. 强时效入口：`/receiver/data`

代码链路：
- `src/core/01-entry/receiver/controller/receiver.controller.ts`
- `src/core/01-entry/receiver/services/receiver.service.ts`

关键点：
- 默认 `useSmartCache = true`（`request.options?.useSmartCache !== false`）。
- 启用时走 `CacheStrategy.STRONG_TIMELINESS`。
- 当前默认强时效 TTL 是 `5s`（来自 SmartCache 配置）。
- 设置 `useSmartCache=false` 时，会直接走原始拉取流程，不走 smart cache 编排。

## 4. 弱时效入口：`/query/execute`

代码链路：
- `src/core/01-entry/query/controller/query.controller.ts`
- `src/core/01-entry/query/services/query-execution-engine.service.ts`

关键点：
- `by_symbols` 主链固定走 `CacheStrategy.WEAK_TIMELINESS`。
- 默认弱时效 TTL 是 `300s`（5 分钟）。
- 更新机制是“读时更新”（read-through），不是定时全量刷新。

当前实现注意事项（很重要）：
- `QueryRequestDto.options.useCache` 当前在执行引擎主链里没有生效分支（不会真正控制是否绕过缓存）。
- `QueryRequestDto.maxAge` 当前用于下游超时换算（`timeout`），不是缓存 TTL 开关。

## 5. support-list 弱时效机制

代码链路：
- `src/core/01-entry/query/controller/support-list-query.controller.ts`
- `src/core/01-entry/query/services/support-list-query.service.ts`
- `src/core/03-fetching/support-list/services/support-list-read.service.ts`
- `src/core/03-fetching/support-list/services/support-list-sync.service.ts`
- `src/core/03-fetching/support-list/services/support-list-sync.scheduler.ts`

机制说明：
- 首次访问或版本异常时，读取前触发 `ensureTypeInitialized`。
- 定时任务按 `SUPPORT_LIST_SYNC_CRON` 刷新（默认每天 2AM）。
- 增量保留窗口 7 天（`SUPPORT_LIST_DELTA_TTL_SECONDS`）。
- `since` 版本链断裂时返回 `409`（`SUPPORT_LIST_RESYNC_REQUIRED`），要求先走一次全量。

## 6. 分时线与流式入口时效

### 6.1 分时线（HTTP）
- 入口：`/api/v1/chart/intraday-line/snapshot`、`/api/v1/chart/intraday-line/delta`
- 实现：`src/core/03-fetching/chart-intraday/services/chart-intraday-read.service.ts`
- 模型：snapshot 使用“`1m` 历史基线 + 实时 `1s` 点”合并；delta 依赖 cursor 增量。
- 这组接口不是按 SmartCache 强/弱 TTL 运行，而是按点位与 cursor 协议运行。

### 6.2 实时流（WebSocket）
- 入口：`/api/v1/stream-receiver/connect`
- 实现：`src/core/01-entry/stream-receiver/gateway/stream-receiver.gateway.ts`
- 流缓存默认值：
  - Hot: `STREAM_CACHE_HOT_TTL_MS=5000`
  - Warm: `STREAM_CACHE_WARM_TTL_SECONDS=300`

## 7. 环境变量覆盖（与时效直接相关）

### 7.1 Smart Cache

| 环境变量 | 默认值 | 说明 |
|---|---:|---|
| `SMART_CACHE_TTL_STRONG_S` | `5` | 强时效 TTL |
| `SMART_CACHE_TTL_WEAK_S` | `300` | 弱时效 TTL |
| `SMART_CACHE_TTL_OPEN_S` | `10` | 开市/近实时 TTL（默认 `strong*2`） |
| `SMART_CACHE_TTL_CLOSED_S` | `1800` | 闭市 TTL |
| `SMART_CACHE_MAX_CONCURRENCY` | 自动计算 | SmartCache 最大并发 |

补充：
- `weekendTtlSeconds` 默认是 `closed*2`（通常 `3600`）。
- 强/弱接口当前分别固定使用 `STRONG_TIMELINESS` / `WEAK_TIMELINESS` 策略。

### 7.2 Stream Cache

| 环境变量 | 默认值 | 说明 |
|---|---:|---|
| `STREAM_CACHE_HOT_TTL_MS` | `5000` | 热层缓存（毫秒） |
| `STREAM_CACHE_WARM_TTL_SECONDS` | `300` | 温层缓存（秒） |
| `STREAM_CACHE_MAX_HOT_SIZE` | `1000` | 热层最大条目 |
| `STREAM_CACHE_BATCH_SIZE` | `200` | 流处理批量 |

### 7.3 support-list

| 环境变量 | 默认值 | 说明 |
|---|---|---|
| `SUPPORT_LIST_SYNC_CRON` | `0 0 2 * * *`（每天 2AM） | 定时同步任务 |
| `SUPPORT_LIST_TYPES` | 内置类型集合 | 要同步的类型 |
| `SUPPORT_LIST_MAX_ITEMS` | `100000` | 单次最大条目 |
| `SUPPORT_LIST_MAX_PAYLOAD_BYTES` | `16777216` | 单次最大载荷字节 |
