# JvQuant 接入 newstockapi 实施指南（WebSocket Only）

本文档说明如何在当前项目中使用 `jvquant` 作为**流式个股行情** Provider。

范围：

- 支持市场：港股（HK）、美股（US）、沪深（SH/SZ）
- 支持能力：`stream-stock-quote`
- 不包含：REST 能力

## 1. 已接入内容

代码已实现以下组件：

1. `JvQuantProvider`
2. `JvQuantStreamContextService`
3. `stream-stock-quote` capability
4. `providersv2` 模块注册与注册表显式注册
5. `jvquant` 的 stream 预设模板（用于生成 `quote_fields` 映射规则）

## 2. 必要环境变量

至少配置：

- `JVQUANT_TOKEN`

可选配置：

- `JVQUANT_SERVER_API`（默认：`http://jvQuant.com/query/server`）
- `JVQUANT_CONNECT_TIMEOUT_MS`（默认：`10000`）

## 3. Provider 名称

使用统一 provider 名称：

- `jvquant`

示例（WebSocket 订阅时指定）：

```json
{
  "symbols": ["00700.HK", "AAPL.US", "600519.SH"],
  "wsCapabilityType": "stream-stock-quote",
  "preferredProvider": "jvquant"
}
```

## 4. 市场与订阅格式

`JvQuantStreamContextService` 会自动把标准符号转换成订阅代码：

1. `00700.HK` -> `lv1_00700`（market=hk）
2. `AAPL.US` -> `lv1_aapl`（market=us）
3. `600519.SH` -> `lv1_600519`（market=ab）
4. `000001.SZ` -> `lv1_000001`（market=ab）

## 5. 连接模型

由于 JvQuant 按市场分配服务器，服务内部采用“单 Provider 多市场连接”模型：

1. `hk` 一条连接
2. `us` 一条连接
3. `ab` 一条连接

对上层仍保持统一 `getStreamContextService()` 接口。

## 6. 数据解析与输出字段

已实现：

1. 二进制 payload 解压（raw deflate）
2. `lv1_*` 行情行解析
3. 统一输出字段：`symbol/lastPrice/volume/turnover/timestamp/tradeStatus`

说明：

- 当前只消费 `lv1`（个股报价）。
- `lv2/lv10` 暂未纳入本次接入范围。

## 7. 启动与验证

启动后建议验证：

1. 日志中出现 `JvQuant WebSocket 连接成功`（对应 hk/us/ab）
2. 使用 `preferredProvider=jvquant` 订阅可收到流数据
3. 广播数据中的 `symbol` 为标准格式（如 `00700.HK`、`AAPL.US`、`600519.SH`）

## 8. 注意事项

1. 若数据库里还没有 `jvquant + stream + quote_fields` 规则，需重启服务让预设模板初始化器执行并生成映射规则。
2. 当前实现是 WebSocket only，不支持通过 `Receiver` 走 REST 查询到 `jvquant`。
3. 若要支持指数/逐笔/十档，需要新增 capability 与解析器分支。
