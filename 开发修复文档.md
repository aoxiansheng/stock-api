## 已完成修复

### E. upstream 本地运行时边界收口

背景：
上一轮把 session 迁到了共享缓存，但跨实例时仍可能因为 owner 迁移或共享 upstream 假设导致旧实例误退订、错误清理或生命周期漂移。

本轮落地：
1. `ChartIntradayStreamSubscriptionService` 保留本地 upstream runtime registry，只让当前实例维护自己的内部订阅、宽限期与 module destroy 清理。
2. `release` 在共享 session 删除后，立即按 `activeSessionCount === 0` 清理共享 upstream 元数据，同时把本地内部订阅回收职责留在本地 runtime。
3. `cleanupExpiredSessions()` 对本地 runtime 做自愈清理，只基于“session 是否仍存在”决定本地回收，不再因远端 touch 抢 owner 而提前误退订。
4. 补充跨实例回归：远端 touch 不应提前释放旧实例本地订阅；远端 release 后旧实例可在 cleanup 周期内回收本地内部订阅。

验收结果：
- 跨实例 touch/release 回归通过。
- `onModuleDestroy` 仅清理本地残留 upstream，不再扫描共享全量 upstream。

### F. shared session 状态并发收口

背景：
共享 session 写路径存在 `touch/bind/release/unbind` 互相覆盖风险，容易把已删除 session 写回或把 `boundClientIds` 更新丢掉。

本轮落地：
1. `ChartIntradaySessionService` 为共享 session 关键写路径加入最小成本并发保护。
2. `createSession / touchSession / bindClientToSession / releaseSession / unbindClientSessions / touchBoundSessionsAsync / unbindClientAsync / deleteUpstream` 统一纳入受保护变更路径。
3. 保留当前 `scan + counter` 结构，避免回退到脆弱的共享数组索引。

验收结果：
- 目标单测全部通过。
- 单文件类型检查通过。
